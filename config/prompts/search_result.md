### 6. 示例处理

#### 输入网页内容示例:
```
"我在使用 Rust 处理向量时遇到了 E0382 错误...
有人建议使用 clone()，但我觉得性能不好...
最后发现可以通过引用传递解决..."

代码示例:
let v = vec![1,2,3];
let v2 = v;  // 所有权转移
println!("{}", v);  // 错误! v 已被移动
```

#### 输出格式示例:
```markdown
# 编译错误: E0382 - 使用被移动的值

## 🚨 错误信息
```rust
error[E0382]: use of moved value: `v`
```

## 🔍 根本原因
向量 `v` 在赋值给 `v2` 时所有权被转移，后续无法再使用。

## 🛠️ 解决方案

### 方案 1: 使用引用传递
```rust
let v = vec![1,2,3];
let v2 = &v;  // 使用引用，不获取所有权
println!("{:?}", v);  // 正常使用
println!("{:?}", v2);
```

**适用场景:** 只需要读取数据，不需要获取所有权
**注意事项:** 确保引用的生命周期足够长

### 方案 2: 使用 clone() 方法
```rust
let v = vec![1,2,3];
let v2 = v.clone();  // 创建数据的完整拷贝
println!("{:?}", v);
println!("{:?}", v2);
```

**适用场景:** 需要独立的数据副本
**注意事项:** 对于大数据量可能影响性能

## 📝 补充说明
- Rust 所有权系统要求每个值只有一个所有者
- 移动语义是 Rust 内存安全的基础
- 引用不会获取所有权，适合只读访问

## 🏷️ 标签
`E0382` `所有权` `移动语义` `初学者`
```
